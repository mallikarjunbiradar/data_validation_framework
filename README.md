# Data Validation Framework

A Python-based data validation framework for profiling, schema validation, Great Expectations checks, and source-vs-sink reconciliation.

## Features

- Data profiling (`profile`) with null/distinct/outlier metrics
- Schema comparison (`schema_check`) from config-driven expectations
- Great Expectations validation (`ge_validate`)
- Record-level reconciliation (`reconcile`) with samples and summaries
- Optional AI-assisted insights (with safe fallback when AI is unavailable)
- JSON report generation by default with timestamped filenames
- CLI and FastAPI interfaces

## Project Structure

- `validators/` - validation modules
  - `dqr_validator.py`
  - `schema_validator.py`
  - `GE_validator.py`
  - `reconciliation_validator.py`
- `utils/` - helper utilities and AI integration
- `config/` - dataset rules/configuration
- `data/` - sample source/sink datasets
- `reports/` - generated output reports
- `cli.py` - command-line entry point
- `api.py` - FastAPI service
- `scripts/sample_api_client.py` - API consumer sample

## Requirements

- Python 3.10+ recommended
- Dependencies in `requirements.txt`

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Optional AI Setup

Set your API key in the same shell that runs CLI/API:

```bash
export OPENAI_API_KEY="your_key_here"
```

Optional timeout override:

```bash
export OPENAI_TIMEOUT_SECONDS=15
```

## CLI Usage

General form:

```bash
python3 cli.py <action> [options]
```

Actions:

- `profile`
- `schema_check`
- `ge_validate`
- `reconcile`

Common options:

- `--config`, `-c` (default: `config/rules.yaml`)
- `--dataset`, `-d` (for dataset-based actions)
- `--file_path`, `-f` (for `profile`)
- `--report_file`, `-r` (optional custom report path)
- `--ai_report` (generate AI summary in report)
- `--ai_model` (override model name)
- `--no_ai` (disable all AI-assisted analysis)

Examples:

```bash
python3 cli.py profile --file_path data/sample_source.csv
python3 cli.py schema_check -d customer_data --ai_report
python3 cli.py ge_validate -d customer_data
python3 cli.py reconcile -d customer_data --no_ai
```

## FastAPI Usage

Start API:

```bash
uvicorn api:app --reload --host 0.0.0.0 --port 8000
```

Open docs UI:

- `http://127.0.0.1:8000/docs`

Endpoints:

- `GET /health`
- `POST /profile`
- `POST /schema_check`
- `POST /ge_validate`
- `POST /reconcile`

Each endpoint supports:

- `config`
- `report_file`
- `ai_report`
- `ai_model`
- `no_ai`

Additional request fields:

- `/profile`: `file_path`
- dataset-based endpoints: `dataset`

## Sample API Client

Use the sample script to call individual operations:

```bash
python scripts/sample_api_client.py health
python scripts/sample_api_client.py profile --ai_report
python scripts/sample_api_client.py schema_check --dataset customer_data
python scripts/sample_api_client.py ge_validate --dataset customer_data
python scripts/sample_api_client.py reconcile --dataset customer_data
python scripts/sample_api_client.py all
```

## Reports

Reports are generated by default in `reports/` using:

`<action>_<dataset_or_file>_<timestamp>.json`

Reconciliation reports are stored in a presentable shape with:

- summary counts
- sample rows
- optional AI insights

## Notes

- If AI fields are blank, check:
  - `OPENAI_API_KEY` is set
  - API key quota/access
  - `no_ai` is `false`
- `ai_status` in API responses helps diagnose AI behavior:
  - `generated`
  - `failed_api`
  - `failed_parse`
